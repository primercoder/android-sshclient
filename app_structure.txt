===============================================================================
  Bug History (用户反馈记录)
===============================================================================

v1 问题: SSH shell模式, broadcast stream数据丢失, 直连失败
  - StreamController.broadcast() 无监听者时丢弃数据 → shell提示符丢失
  - 直连和收藏路径不一致 → singleton provider 状态污染
  - 解决: 改用 exec 模式, connect()前先disconnect()

v2 问题: Utf8Decoder类型转换, 路径跟踪, 退出逻辑
  - utf8.decoder as StreamTransformer<Uint8List, String> → 类型转换崩溃
  - cd Desktop && ls → 路径变成 "Desktop && ls" (本地字符串解析)
  - 退出对话框选项逻辑错误
  - 解决: utf8.decodeStream(), 每条命令包装 cd <pwd> && <cmd> && pwd

v3 问题: 主机覆盖, 扫描UI, SCP错误, 文件下载
  - hostId用IP导致第二个同IP主机覆盖第一个
  - Utf8Decoder SCP下载崩溃
  - 下载文件不存在 (直接写 /storage/emulated/0/Download/ 被SAF拦截)
  - 解决: UUID, FilePicker.saveFile(SAF)

v4 问题: cd包装后下拉目录仅显示'..', 会话不保持, 删除无效
  - ls -a 用 test -d 判断目录 → 多次SSH调用且不准确
  - 保持会话后重新进入 → 新session, 对话清空
  - 历史/主机删除无响应
  - SCP失败连带SSH传输关闭 (Transport is closed)
  - 下载无本地路径输入框
  - 解决: ls -la 解析权限位'd'判断目录, ChatNotifier._activeSessions,
    不显式 scpSession.close(), 双输入框(远程+本地), 已测通

一、技术栈
  Flutter 3.41.9 | Dart 3.11.5 | dartssh2 2.17.1 | Riverpod 2.6.1
  最低 Android API 34 | Target API 35 | SQLite (sqlite3)

  │  ├── (+) FAB: 添加收藏主机                                   │

三、命令执行引擎
  每条命令包装为: cd "<当前pwd>" && <用户命令> && pwd
  输出解析: 末行 = 新pwd (更新顶部栏, 不显示在对话)
           其余行 + stderr = 命令输出 (显示在对话)

四、路径下拉 (目录浏览)
  点击路径栏 → execute('cd "<pwd>" && ls -la && pwd')
  解析: 权限位首字符='d' → 目录 → 加入列表
  排序: '..' 最前 → 非隐藏目录(字母序) → 隐藏目录(字母序)
  点击目录 → cd "<dir>" + 更新pwd

五、会话生命周期
  ChatNotifier._activeSessions: Map<sessionId, Session>
  - 进入: startNewSession(hostId) → 复用活跃会话或新建
  - 保持会话: 不调用endSession → _activeSessions保留
  - 主页断开: endSession(hostId) + sshConnection.disconnect()
  - 会话ID: generateId(hostName, hostIp, timestamp)
  - endTime设入DB后变为历史记录
  - 复用会话: 不显示"连接已建立", 通过cd "$lastDir" && pwd恢复pwd

六、文件传输 (SCP)
  上传: scp -t 协议 (StreamIterator增量读ack)
  下载: SSH exec cat 读取远程文件 (避免SCP -f双向协议死锁)
  下载路径: 优先用户配置目录, 回退app私有目录
  错误: 通过\001 + msg检测SCP错误, 不读取stderr(避免死锁)
  删除不完整文件: 下载失败时close sink + delete

七、SSH会话管理
  连接: SSHSocket.connect() → SSHClient() → authenticate
  命令: client.execute(cmd, pty) → 新channel → utf8.decodeStream
  命令(stderr): executeCombined() 同时收集stdout + stderr
  文件: client.execute('scp -t/cat ...') → 新channel
  断开: client.close() → client=null → activeConnection=null

八、局域网扫描
  CIDR解析: /16 ~ /30, 排除网络地址和广播地址
  detectCurrentCidr: 读取/proc/net/route → NetworkInterface → 默认/24
  批量扫描: 100个Socket.connect并发/批, 3秒批次间隔
  暂停/继续: ScanAbort (Completer机制)
  停止: ScanAbort.stop() 跳出循环
  状态: ScanState (idle/scanning/paused) 控制UI动画

九、快捷命令
  quick_commands表存储 label + command + sortOrder
  内置种子命令 (is_builtin=0, 可自由编辑删除)
  ⋮按钮 → 管理面板 (ReorderableListView, 拖拽排序)
  添加/编辑: 对话框输入显示名称 + 命令
  删除: 无确认直接删除

十、主题
  亮蓝 #2196F3 (light) / #64B5F6 (dark)
  Material3, colorSchemeSeed
  深色模式切换 (Provider<bool>)

十一、数据库表 (SQLite)
  hosts: host_id(UUID PK), display_name, current_ip, port,
         username, password, mac_address, host_key_fingerprint, ...
  sessions: session_id(PK), host_id, host_name, host_ip,
            start_time, end_time, command_count, last_working_dir
  chat_messages: message_id(PK), session_id(FK), type, content, ...
  transfer_tasks: transfer_id(PK), session_id(FK), direction, ...
  quick_commands: command_id(PK AUTO), label, command, sort_order, ...

十二、关键Provider
  sshClientServiceProvider    → SshClientService (singleton)
  sshConnectionProvider       → SshConnectionNotifier (state machine)
  chatProvider                → ChatNotifier (messages + sessions)
  transferProvider            → TransferNotifier (file transfers)
  hostDaoProvider             → HostDao (async)
  sessionDaoProvider          → SessionDao (async)
  messageDaoProvider          → MessageDao (async)
  quickCommandDaoProvider     → QuickCommandDao (async)
  isDarkModeProvider          → bool
  downloadDirProvider         → String (下载目录)
