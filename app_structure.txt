一、核心流程设计
================================================================================

  主页 (主机收藏列表)
  ├── 收藏的主机卡片列表 (Host对象: displayName, ip, port, username, password, fingerprint)
  │   └─ 点击卡片 → [1]建立SSH连接 → [2]连接成功 → [3]跳转会话页(可操作)
  │                              → [2]连接失败 → 弹窗报错, 不跳转
  ├── [+] FAB: 添加收藏主机 (仅保存信息到DB, 不连接)
  │   └─ 弹窗: IP / 端口 / 用户名 / 密码 / 备注 → 保存 → 主页刷新
  └── [直连] 按钮: 手动输入凭据并立即连接 (不保存)
      └─ 弹窗: IP / 端口 / 用户名 / 密码 → 点击连接
          ├─ [1]建立SSH连接 → [2]成功 → 跳转会话页
          └─ [2]失败 → 弹窗报错

  会话页 (聊天式SSH)
  ├─ 只读模式 (历史回放): 从DB读取消息, 底部显示"只读模式"
  └─ 活动模式 (实时SSH):
      ├─ 顶部: 路径指示器 (点击显示目录列表)
      ├─ 中间: 消息列表 (命令/输出/系统)
      ├─ 快捷命令Chips
      └─ 输入栏 + 文件面板

二、SSH 连接引擎 (修复要点)
================================================================================

  问题诊断:
  ┌─────────────────────────────────────────────────────────────────────┐
  │ 当前代码的问题:                                                      │
  │ 1. ChatPage._connect() 不等待连接完成就允许用户输入, 导致发命令时   │
  │    连接可能还没建立或已失败                                         │
  │ 2. 发送命令 → 但shell的输出流没有被正确捕获展示:                    │
  │    a. sendShellCommand() 只写入stdin, 不等待/捕获输出              │
  │    b. _outputController 虽然监听了stdout, 但chat_page中订阅可能     │
  │       存在时序问题 (建立连接后才订阅, 错过shell prompt等)           │
  │ 3. 连接失败后仍然跳转到会话页, 用户看到空白页面                     │
  └─────────────────────────────────────────────────────────────────────┘

  修复方案:
  ┌─────────────────────────────────────────────────────────────────────┐
  │ 连接流程:                                                           │
  │  1. 用户在主页触发连接(收藏卡片点击/直连按钮)                       │
  │  2. 显示 LoadingDialog "正在连接..."                                │
  │  3. SshClientService.connect():                                      │
  │     a. TCP连接 (10秒超时)                                            │
  │     b. SSH握手 + 主机密钥验证                                        │
  │     c. 密码认证                                                      │
  │     d. 打开 Shell channel + PTY                                      │
  │     e. 订阅 stdout → _outputController                               │
  │     f. 启动 keepalive                                                │
  │  4. 连接成功 → 关闭LoadingDialog → Navigator.push(会话页)          │
  │  5. 连接失败 → 关闭LoadingDialog → 弹窗显示错误原因                 │
  │                                                                      │
  │ 命令执行流程:                                                        │
  │  1. 用户输入命令, 按发送                                             │
  │  2. 创建 ChatMessage 写入DB (立即显示"已发送")                      │
  │  3. sshService.sendShellCommand(cmd) → 写入stdin                    │
  │  4. Shell处理命令, 输出到stdout                                      │
  │  5. _outputSubscription 收到数据 → chat.addOutput() → DB存储+UI显示 │
  │  6. 输出流持续, 直到命令完成 (解析shell提示符可判断命令结束)        │
  └─────────────────────────────────────────────────────────────────────┘

  改进: 会话页增加连接状态指示:
  - 连接中: 显示 Loading 动画, 输入栏禁用
  - 已连接: 输入栏可用, 显示"已连接"系统消息
  - 连接失败: 显示错误消息, 输入栏禁用, 底部显示"重试"按钮

三、Android 本地子网检测
================================================================================

  Android 平台限制:
  - dart:io 的 NetworkInterface.list() 在 Android上不可靠
  - InternetAddress 不提供 netmask
  - 需要读取 Android 系统文件 /proc/net/route 获取网关和掩码

  实现方案 (多层降级):
  ┌─────────────────────────────────────────────────────────────────────┐
  │ 第一层: 读取 /proc/net/route (Android 可用)                         │
  │   文件格式: Iface, Destination, Gateway, Flags, RefCnt, Use,       │
  │            Metric, Mask, MTU, Window, IRTT                         │
  │   - 解析 Destination(目标网络) + Mask(子网掩码)                    │
  │   - 转换为 CIDR 格式: ip/maskBits                                   │
  │   示例:                                                             │
  │   wlan0  00000000  0108A8C0  0003  0  0  0  00FFFFFF  0  0  0     │
  │   → Destination=0.0.0.0(默认路由), Gateway=192.168.8.1            │
  │   → Mask=00FFFFFF → 255.255.255.0 → /24                           │
  │                                                                     │
  │ 第二层: NetworkInterface.list() (Android 10+)                      │
  │   获取第一个非回环 IPv4 地址                                        │
  │   默认使用 /24 (因为无法获取 netmask)                               │
  │                                                                     │
  │ 第三层: 回退到 192.168.1.1/24                                       │
  │                                                                     │
  │ 用户的输入始终可编辑                                                   │
  └─────────────────────────────────────────────────────────────────────┘

  /proc/net/route 解析核心逻辑:
    String destinationHex, maskHex;
    读取 /proc/net/route 的每一行 (跳过标题行)
    找到 Iface 非空且 Destination 非 00000000 的行
    将 Destination 的 hex 转为 IP (小端序)
    将 Mask 的 hex 转为 IP, 计算掩码位数

四、收藏主机模型
================================================================================

  Host 模型需要存储连接凭据:
  - hostId: SSH指纹 (TEXT PK)
  - displayName: 用户自定义名称 (TEXT)
  - currentIp: IP地址 (TEXT)
  - port: SSH端口 (INT)
  - username: SSH用户名 (TEXT)
  - password: SSH密码 (TEXT)  — 新增字段
  - macAddress: MAC地址 (TEXT, nullable)
  - hostKeyFingerprint: SSH主机指纹 (TEXT)
  - hostKeyAlgorithm: 密钥算法 (TEXT, nullable)
  - sshBanner: SSH版本信息 (TEXT, nullable)
  - firstSeenAt: 首次添加时间 (TEXT)
  - lastSeenAt: 最后连接时间 (TEXT)
  - connectionCount: 连接次数 (INT)
  - notes: 备注 (TEXT, nullable)
